.功能需求

2.1.1 商品管理模块

 商品搜索：支持按商品名称、分类、价格区间、发布者等条件搜索商品

 商品列表：展示商品基本信息，包括名称、价格、分类、成色、发布状

态等

 商品详情：查看商品详细信息，包括实物图、商品描述、卖家信息、发

布时间等

 商品分类：支持按分类浏览商品，如数码产品、书籍教材、生活用品、

服饰鞋包等

 商品管理：支持发布、编辑、上下架、删除自有商品

2.1.2 用户管理模块

 用户注册：新用户账号注册、基础信息与实名认证录入

 用户查询：管理员按学号 / 工号、姓名等条件查询用户信息

 用户详情：查看用户基本信息、发布历史、交易记录、信誉评分

 用户状态：显示用户当前账号状态、交易权限与违规记录

2.1.3 交易订单模块

 下单交易：买家提交订单，生成交易记录

 订单流转：待付款、待发货、待收货、已完成、已取消全状态管理

 订单记录：查看个人历史交易订单，支持按状态、时间筛选

 逾期提醒：显示待付款、待确认收货的超时订单

2.1.4 统计分析模块

 交易统计：统计平台成交量、交易额、交易趋势、热门商品分类

 用户统计：统计用户活跃度、商品发布量、交易参与度

 商品统计：统计商品分类分布、上架总量、成交转化率

 数据可视化：通过图表展示各项统计数据

2.1.5 系统管理模块

 用户登录：支持不同角色登录（管理员、普通用户）

 权限管理：基于角色的权限控制

 内容审核：管理员审核违规商品、处理交易纠纷

 系统设置：配置平台交易规则、违规处罚机制等

 数据备份：支持数据导出和备份

3.设计风格

 整体风格：简洁、清爽、易用，符合校园年轻用户审美

 色彩方案：以青绿色与白色为主，体现活力与信任感

 布局：响应式设计，适配不同设备屏幕

 导航：清晰的导航结构，便于快速访问各功能模块

4.页面设计

 登录页面：简洁的登录注册表单，支持账号密码登录

 首页：平台推荐商品、快速搜索、常用功能入口

 商品列表页：分页显示商品列表，支持排序和多条件筛选

 商品详情页：详细展示商品信息，提供下单与收藏按钮

 用户中心页：个人信息管理、我的商品、我的订单入口

 订单管理页：订单列表展示，支持状态流转操作

 统计分析页：图表展示统计数据，支持时间范围选择

2.2 考点说明

一、Spring Boot 核心基础

自动配置原理：起步依赖 Starter 机制、自动装配流程、条件注解应用

IOC/DI 与 Bean 管理：注解创建 Bean、依赖注入、Bean 作用域与生命周期

配置管理：application.yml/properties 配置文件、多环境 profile 切换、

自定义配置属性

项目启动机制：启动类注解、内嵌 Servlet 容器、项目启动流程

二、Spring Boot Web 开发

RESTful 接口设计：请求规范、路径设计、HTTP 方法语义

请求与参数处理：常用请求注解、多类型参数绑定（基本类型、实体类、集合、

日期）

响应处理：JSON 统一返回封装、全局异常处理、跨域配置

拦截器与过滤器：登录权限拦截、请求日志埋点、跨域处理

文件操作：商品图片上传与下载接口实现

三、数据持久层整合

MyBatis/MyBatis-Plus 整合：配置方式、Mapper 扫描、基础 CRUD 封装

参数与结果映射：#{} 与 ${} 区别、结果集映射、多表关联查询

动态 SQL：条件判断、循环标签、SQL 片段复用

分页与主键：分页插件集成、主键生成策略、批量操作

缓存机制：一级 / 二级缓存原理、拓展 Redis 缓存方案

四、企业级特性与项目整合

事务管理：ACID 特性、声明式事务注解、事务传播行为与隔离级别

AOP 面向切面：统一日志处理、接口性能监控、业务逻辑增强

分层架构：Controller 表现层、Service 业务层、Mapper 持久层三层规范

项目打包部署：Jar 包打包方式、内嵌容器运行、生产启动配置

五、综合应用

完整业务 CRUD：单表与多表联动的业务逻辑实现

分页查询：分页插件集成与分页结果标准化封装

登录鉴权：会话控制、权限校验、接口访问拦截

前后端联调：接口规范对齐、参数联调、异常场景适配
