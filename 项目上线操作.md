\# CakeShop 蛋糕系统 - 上线操作文档



\---



\## 📋 文档说明



本文档详细介绍如何将 CakeShop 蛋糕系统部署到本地 Docker Desktop 和阿里云服务器。适合 Docker 和阿里云零基础的用户参考。



\*\*项目技术栈：\*\*

\- 后端：Spring Boot 2.2.6

\- 数据库：MySQL 8.0+

\- 构建工具：Maven 3.5+

\- 前端：Thymeleaf + HTML + CSS



\---



\## 🚀 第一部分：本地 Docker Desktop 部署



\### 1.1 环境准备



\#### 1.1.1 安装 Docker Desktop



\*\*步骤：\*\*

1\. 访问 Docker 官方网站：https://www.docker.com/products/docker-desktop/

2\. 下载适合您操作系统的版本（Windows/macOS）

3\. 双击安装包进行安装

4\. 安装完成后启动 Docker Desktop

5\. 等待 Docker 启动完成（右下角图标变绿）



\*\*验证安装：\*\*

打开命令行终端，执行以下命令：

```bash

docker --version

docker-compose --version

```

如果显示版本信息，说明安装成功。



\#### 1.1.2 准备项目文件



确保您的项目目录结构如下：

```

CakeShop/

├── src/

├── pom.xml

├── README.md

└── (其他项目文件)

```



\### 1.2 创建 Docker 配置文件



在项目根目录创建以下文件：



\#### 1.2.1 创建 Dockerfile



\*\*文件路径：\*\* `CakeShop/Dockerfile`



```dockerfile

\# 使用 Java 8 基础镜像

FROM openjdk:8-jdk-alpine



\# 设置工作目录

WORKDIR /app



\# 复制 Maven 构建后的 JAR 文件

COPY target/CakeShop-0.0.1-SNAPSHOT.jar app.jar



\# 暴露端口

EXPOSE 8090



\# 设置环境变量

ENV SPRING\_PROFILES\_ACTIVE=prod



\# 启动应用

ENTRYPOINT \["java", "-jar", "app.jar"]

```



\#### 1.2.2 创建 docker-compose.yml



\*\*文件路径：\*\* `CakeShop/docker-compose.yml`

\### 1.3 构建项目



\*\*步骤：\*\*

1\. 打开命令行终端

2\. 进入项目根目录：

```bash

cd C:\\maven\\CakeShop\\CakeShop

```

3\. 使用 Maven 构建项目：

```bash

mvn clean package -DskipTests

```



\*\*等待构建完成\*\*，构建成功后会在 `target` 目录生成 `CakeShop-0.0.1-SNAPSHOT.jar` 文件。



\### 1.4 使用 Docker Compose 启动服务



\*\*步骤：\*\*

1\. 确保 Docker Desktop 正在运行

2\. 在项目根目录执行：

```bash

docker-compose up -d

```



\*\*命令说明：\*\*

\- `docker-compose up`：启动所有服务

\- `-d`：后台运行模式



\### 1.5 验证部署



\*\*步骤：\*\*

1\. 查看容器状态：

```bash

docker-compose ps

```

2\. 查看日志：

```bash

docker-compose logs -f

```

3\. 访问应用：

&#x20;  - 打开浏览器

&#x20;  - 访问：http://localhost:8090

&#x20;  - 访问管理员后台：http://localhost:8090/admin

\## ☁️ 第二部分：阿里云服务器上线



\### 2.1 准备工作



\#### 2.1.1 注册阿里云账号



\*\*步骤：\*\*

1\. 访问阿里云官网：https://www.aliyun.com/

2\. 注册账号并完成实名认证

3\. 登录阿里云控制台



\#### 2.1.2 购买云服务器 ECS



\*\*步骤：\*\*

1\. 登录阿里云控制台

2\. 在产品列表中找到「云服务器 ECS」

3\. 点击「立即购买」

4\. 配置服务器参数：

&#x20;  - \*\*地域\*\*：选择离您用户最近的地域（如上海、北京）

&#x20;  - \*\*实例规格\*\*：入门级选择「突发性能型 t6.large.2」或「共享型 s6.large.2」

&#x20;  - \*\*操作系统\*\*：选择「CentOS 7.9 64位」

&#x20;  - \*\*存储\*\*：系统盘 40GB 高效云盘

&#x20;  - \*\*带宽\*\*：按实际需求选择，建议至少 2Mbps

&#x20;  - \*\*购买时长\*\*：按需或包年包月



5\. 设置登录密码（重要！记住密码）

6\. 完成购买



\#### 2.1.3 配置安全组



\*\*步骤：\*\*

1\. 在 ECS 控制台找到「安全组」

2\. 点击「配置规则」

3\. 添加以下端口规则：

&#x20;  | 端口 | 协议 | 来源 | 用途 |

&#x20;  |------|------|------|------|

&#x20;  | 22 | TCP | 0.0.0.0/0 | SSH 远程连接 |

&#x20;  | 80 | TCP | 0.0.0.0/0 | HTTP 访问 |

&#x20;  | 443 | TCP | 0.0.0.0/0 | HTTPS 访问 |

&#x20;  | 8090 | TCP | 0.0.0.0/0 | 应用端口 |

&#x20;  | 3306 | TCP | 0.0.0.0/0 | MySQL 端口（可选）|

\*\*安装 Nginx：\*\*

```bash

systemctl start nginx

systemctl enable nginx

```



\*\*配置反向代理：\*\*

```bash

vi /etc/nginx/conf.d/cakeshop.conf

```



添加以下内容：

```nginx

server {

&#x20;   listen 80;

&#x20;   server\_name your-domain.com www.your-domain.com;



&#x20;   location / {

&#x20;       proxy\_pass http://localhost:8090;

&#x20;       proxy\_set\_header Host $host;

&#x20;       proxy\_set\_header X-Real-IP $remote\_addr;

&#x20;       proxy\_set\_header X-Forwarded-For $proxy\_add\_x\_forwarded\_for;

&#x20;       proxy\_set\_header X-Forwarded-Proto $scheme;

&#x20;   }



&#x20;   # 静态资源缓存

&#x20;   location \~\* \\.(js|css|png|jpg|jpeg|gif|ico|svg)$ {

&#x20;       expires 7d;

&#x20;       add\_header Cache-Control "public, no-transform";

&#x20;       proxy\_pass http://localhost:8090;

&#x20;   }

}

```



\*\*重启 Nginx：\*\*

```bash

nginx -s reload

```

```nginx

&#x20;   server\_name your-domain.com www.your-domain.com;

&#x20;   listen 443 ssl;

&#x20;   server\_name your-domain.com www.your-domain.com;



&#x20;   ssl\_certificate /etc/nginx/certs/your-cert.crt;

&#x20;   ssl\_session\_timeout 1d;

&#x20;   ssl\_session\_cache shared:SSL:50m;

&#x20;   ssl\_protocols TLSv1.2 TLSv1.3;

&#x20;   ssl\_ciphers HIGH:!aNULL:!MD5;



&#x20;   location / {

&#x20;       proxy\_pass http://localhost:8090;

&#x20;       proxy\_set\_header Host $host;

&#x20;       proxy\_set\_header X-Forwarded-For $proxy\_add\_x\_forwarded\_for;
```

\## 📞 常见问题排查



\### Q1：Docker Compose 启动失败



\*\*可能原因：\*\*

\- 端口被占用

\- MySQL 初始化失败

\- 网络问题



\*\*解决方法：\*\*

```bash

\# 查看日志

docker-compose logs



\# 检查端口占用

netstat -tlnp | grep 8090



\# 删除容器重新启动

docker-compose down

docker-compose up -d

```



\### Q2：无法访问应用



\*\*可能原因：\*\*

\- 安全组未开放端口

\- 防火墙阻止

\- 服务未启动



\*\*解决方法：\*\*

```bash

\# 检查安全组配置

\# 检查防火墙

firewall-cmd --list-all



\# 查看容器状态

docker-compose ps

```



\### Q3：数据库连接失败



\*\*可能原因：\*\*

\- 数据库未启动

\- 连接配置错误

\- 密码错误



\*\*解决方法：\*\*

```bash

\# 检查数据库容器

docker-compose ps mysql



\# 进入数据库容器

docker exec -it cakeshop-mysql mysql -u admin -padmin123



\# 检查连接配置

cat docker-compose.yml | grep SPRING\_DATASOURCE

```

\*\*可能原因：\*\*

\- 服务器未启动

\- 网络问题



\*\*解决方法：\*\*

1\. 在阿里云控制台检查安全组配置

2\. 确认服务器已启动

3\. 尝试使用其他网络连接
